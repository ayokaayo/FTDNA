## What This Repo Is

A shared skill library and automation foundation for Miguel Angelo's team. Skills are Claude instruction packages that can be triggered on demand, composed into workflows, or run autonomously by agents.

The repo serves three audiences:
1. **Humans** — team members who prompt Claude, answer a few questions, and get a skill created or executed
2. **Agents** — autonomous or semi-autonomous Claude instances that navigate the registry to find and chain skills
3. **The skill-architect** — a meta-skill that creates other skills following quality standards

## Key Decisions (2026-03-13)

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Repo name | `automate` | Broader than "skills" — covers future agentic workflows |
| Registry depth | Rich | name, description, path, author, domain, autonomy, inputs/outputs, complexity, status. Auto-generated by skill-architect. |
| Versioning | Git-managed | No semver in registry. Git history handles versioning. Simple overwrite model. |
| Schema now vs. later | Lightweight schema now | Thin registry + folder conventions. Cheap to set up, expensive to retrofit later. |
| Skill-creator relationship | Compose, don't replace | skill-architect handles craft/writing. skill-creator handles eval/iterate loop. They compose. |
| Domain encoding | Generic | Not product-design-specific. Encodes meta-skill of writing good instructions. Domain-agnostic. |
| Autonomy target | Ship-ready by default | Skills should work without human review. Review happens after, not as a gate. |
| User base | Product/Design team first | Skill-architect should be approachable for non-technical users. |

## Registry Schema (Rich)

Each skill entry contains:
- `name` — kebab-case identifier (matches folder name)
- `path` — relative path from repo root
- `description` — what it does + when to trigger (pushy)
- `author` — creator/owner
- `domain` — tags like "product-management", "engineering", "documentation"
- `autonomy` — guided | semi-auto | autonomous
- `inputs` — what it expects (user-prompt, files, URLs, etc.)
- `outputs` — what it produces (.md, .docx, clickup-task, etc.)
- `complexity` — simple | moderate | complex
- `created` / `updated` — dates
- `status` — draft | active | deprecated

Future addition when 10+ skills exist: `composes_with` field for skill-to-skill relationships.
